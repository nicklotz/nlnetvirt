TCP/IP Model

  • Simplifies to 4 layers
  • Typical of "real world" networking

TCP Layers

Application layer

  • Acts as the interface between the end-user and the network
  • Protocols: HTTP(S), FTP, SMTP, DNS


Transport layer

  • Ensures that data is transferred reliably, in sequence, without errors
  • Protocols: TCP, UDP

TCP Layers (Cont.)

Network layer

  • Concerned mostly with packet forwarding
  • Protocols: IP, ICMP, ARP


Data link layer

  • Deals with physical transmission of data
  • Protocols: Ethernet, WiFi, PPP

IP Addressing

IP Addressing: Intro

  • Mechanism assigning unique ID to each device on a network

  • Addresses then used to route traffic across internet and within local networks

  • Two protocols in use today: IPv4 and IPv6

IPv4


  • 32 bits, with each byte separated by a dot (e.g. 192.168.1.1)

  • Used to have classes of ranges, now CIDR predominates

  • Static and dynamic addresses

  • Public and private addresses

How many possible IPv4 addresses are there? Is it enough?

Network Address Translation (NAT)

  • Alleviates IPv4 exhaustion

  • Multiple devices on private network can share single public IP address

  • Static, dynamic, port address translation

  • Used everywhere in virtualization (think about accessing a VM from outside host)

CIDR

  • Classless inter-domain routing
  • Uses subnetting to be more flexible than previous class-based system
  • Combines multiple IP ranges via aggregation - simplifies routing tables
  • Denoted by IP address followed by subnet mask (e.g. 192.168.0.0/24)

Subnetting

  • Divides large network into smaller logical networks

  • Subnet mask separates network from host component of address

  • Can isolate traffic without needing separate physical network

  • Crucial for multi-tenant architectures

Exercises

CIDR calculations

CIDR Exercise 1

CIDR block calculation

Given the CIDR block 192.168.1.0/24, calculate the following:


  • The subnet mask

  • The number of usable IP addresses

  • The range of IP addresses within the block

Answers to CIDR Exercise 1

  • Subnet mask: 255.255.255.0

  • Usable IP addresses: 254
    (256 total addresses minus 2 for the network and broadcast addresses)

  • Range of IP addresses: 192.168.1.1 to 192.168.1.254

CIDR Exercise 2

Subnetting with CIDR

Given a requirement to create four subnets each supporting at least 50 devices, determine the appropriate subnet size and CIDR notation for these subnets if you are starting with a fresh allocation. Assume you are working within a 192.168.0.0 network.

Answer to CIDR Exercise 2

2^6 = 64 and 2^5 = 32

So for 50 hosts per subnet we need at least 6 bits for host addresses

IP addresses are 32 bits, the host portion is 6 bits, so the subnet mask is 26 bits


There can be four 8-bit ranges in an 8-bit octet


The subnets therefore are:

Subnet 1: 192.168.0.0/26

Subnet 2: 192.168.0.64/26

Subnet 3: 192.168.0.128/26

Subnet 4: 192.168.0.192/26

CIDR Exercise 3

CIDR Aggregation

Assume you have the following IP address ranges that need to be aggregated into a single CIDR block:

  • 192.168.1.0/24
  • 192.168.2.0/24
  • 192.168.3.0/24


All addresses use a subnet mask of 255.255.255.0.


What is the smallest single CIDR block that can encompass all these addresses?

Answer to CIDR Exercise 3

We need to hold addresses from 192.168.1.0 to 192.168.3.255


In binary, that's 192.168.00000001.x, 192.168.00000010.x, and 192.168.00000011.x


That's 10 bits worth of host addresses, so the subnet mask is 32 bits - 10 bits = 22 bits


The CIDR block is therefore 192.168.0.0/22

IPv6

  • Developed by the IETF to address the impending exhaustion of IPv4 addresses
  • 128-bit instead of 32-bit, means 2^128 possible addresses
  • Eight groups of four hex digits
    • Can ommit leading zeroes and use double colons :: for consecutive zeroes
  • Unicast, multicast, and anycast addresses
  • IPv6 autoconfiguration

DHCP

  • Dynamic Host Configuration Protocol
  • Server dynamically assigns an IP address to each device on a network

Inter/Intra Domain Routing

IGP and BGP

  • Interior Gateway Protocols (IGP) within a single domain or autonomous system

  • Border Gateway Protocol between autonomous systems

  • Ingress/egress vs inboud/outbound

  • Internal routing generally managed via OSPF or IS-IS

  • ISPs use BGP for peering with other ISPs

  • Both are used in cloud data centers

Software-Defined Networking

Intro to SDN

  • Decouples control plane from data plane
  • Introduces central controller to actively manage network in real time
  • Northbound communication between controller and applications via API's
  • East-west communication between controllers (e.g. OVSDB)
  • Southbound communication between controller and data plane (OpenFlow)

OpenFlow

What is OpenFlow?

  • Primary SDN southbound protocol
  • "Nervous system" between SDN brain and network devices
  • Specifies how controller can modify network device flow entries
  • Challenges: scalability, security, interoperability

Mininet

Mininet Topologies

  • Topologies are the arrangement of network elements (nodes) and the connections between them (links)

  • In Mininet, defined programatically using Python

  • OOTB topologies include linear, tree, single

  • Example: sudo mn --topo linear,3

  • Custom: sudo mn --custom script.py --topo topology_name

  • Helpful Mininet CLI commands for troubleshooting: node, net, dump, link

Routing Control Platforms

RCPs and ONIX

  • RCPs focus on centralization, abstraction, and programmability
  • ONIX is a control platform ideal for distributed networking controllers

  • Uses NIB as a distributed datastore to maintain netowrk state

  • Useful for aggregating multiple virtual networks with customized topologies

4-D Networking

"The 4 Ds"

  • Simplifying networking to four dimensions to improve manageability

  • Decision plane: where centralized control logic resides

  • Dissemination plane: distributes Decision-plane's decisions across the network

  • Discovery plane:Ensures network's current state is accurately known and that changes in the network are quickly detected and reported

  • Data plane:Operates at network device level, handles actual movement of packets across network

Applications

Multi-Tenant Datacenters

  • Different groups of users (tenants) have different access, permission, and QOS requirements

  • Need strict tenant (read: subnet) isolation

  • Increasing use of microsegmentation at the workload level, plus granular RBAC

  • Increasing use of containerized workloads (Docker, Kubernetes) for application isolation

  • Tools like Istio (service mesh) and zero trust networking

Brief Overview of Data Plane

  • Synonymous with forwarding plane in SDN

  • Answers "what do we do with this packet that's come in"?

  • Performs packet processing (encapsulation, decapsulation, error checking) as well as QoS

  • In traditional networks, tightly coupled with control plane, and often same physical device

  • In SDN, decoupled and receives programmatic directions from control plane (which direction again?)

  • A goal of network automation is not needing to directly interact with the data plane

Network Verification

Configuration Verification

  • Ensuring everything is set up and working as intended

  • For data plane, includes correct forwarding, security policy implementation, and meeting performance benchmarks

  • For control plane, includes correct topology mapping, avoiding configuration drift, and securing the control plane itself

  • Ad hoc tools include wireshark, packet tracers, traffic analyzers

  • Going forward, start treating networks like software development: config management (e.g. Ansible, Chef) and GitOps workflows to prevent drift

Networking in AWS

AWS VPCs

  • Virtual Private Cloud: Isolated virutal network for AWS users
  • Includes its own private IP ranges, routing tables, and access control just for you
  • API-driven, and programatically defined via CLI, SDK, other AWS services, or third party tools

Regions and Availability Zones

  • Region: Physical location where AWS data centers live (e.g. us-east-1)
  • Availability Zone: Data center (or cluster of DCs) in a region with redundant connectivity (e.g. us-east-1a)
  • Generally HA between AZs in same region, and DR between regions
  • AWS services and pricing can differ dramatically between regions
